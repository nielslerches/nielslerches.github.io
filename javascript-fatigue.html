JavaScript fatigue

Fri 22 October 2021
By Niels Lerche Sørensen
Category: misc

Do you suffer from JavaScript fatigue? I know that I do (sometimes.)

I think what JavaScript fatigue describes is the feeling of dread that people get when trying to write modern web applications with modern JavaScript-based technologies.

Why do people get this feeling of dread?

I think it's because people (myself included) get tired of "having" to learn yet another technology which the creators pitch as the "one solution to rule them all."

However I think what underlies this feeling if the fact that people (again myself included) know that you don't need something like React, Vue og Svelte to solve really simple problems, like having a modal appear and disappear when a person activates a button.

I've also gotten really tired with the fact that to do anything modern with JavaScript you have to invest a lot of time into a building toolchain for JavaScript, like Webpack. To me, when using these tools it doesn't feel like I'm solving a problem; it just feels like I'm creating one and setting myself up for failure.

I used to be really excited about new JavaScript frameworks; it seemed like they were all going to solve my problems when developing frontend applications.

Now I usually stick to simple scripts on pages, when I can. In fact I'm trying to pull React out of our Admin application for Relevance.
